# Сборка Cube Dodge для Android через GitHub Actions

Этот комплект собирает отдельный APK для Android из исходника `upload/main(1).cpp`.
Локально устанавливать Android Studio, SDK или NDK не нужно: инструменты скачает
GitHub Actions.

## Что получится

- APK для `arm64-v8a`, подходящий для Redmi Note 13;
- горизонтальный полноэкранный режим;
- игровое поле остаётся 800x600 и аккуратно масштабируется на экран телефона;
- игра работает с ограничением 60 FPS даже при включённых 120 Гц;
- сенсорные кнопки не добавляются;
- при потере жизни включается короткая вибрация, при ручной ульте — сильная волна;
- сначала используется вибрация подключённого геймпада, а если Android не
  показывает его моторы приложению — вибрация телефона;
- при USB-подключении DualSense лёгкое сопротивление L2/R2 показывает готовую
  ульту, а при её активации срабатывает более сильный эффект с продавливанием;
- меню настроек открывается кнопкой **Options** и не ставит игру на паузу;
- ускорение по умолчанию выключено: после включения в настройках оно работает,
  пока одновременно удерживаются **L1 + R1**;
- исходник по-прежнему можно собирать и запускать на ПК через VS Code.

## Как загрузить проект

1. Распакуйте архив `CubeDodge-GitHub-Actions.zip` в отдельную папку.
2. Войдите на https://github.com и нажмите **New repository**.
3. Назовите репозиторий, например `cube-dodge-android`.
4. Выберите **Private** или **Public**. Для сборки подходит любой вариант.
5. Не добавляйте README, `.gitignore` или лицензию и создайте пустой репозиторий.
6. На странице пустого репозитория нажмите **uploading an existing file**.
7. Перетащите всё содержимое распакованной папки. Особенно важно, чтобы в
   репозиторий попал файл `.github/workflows/build-apk.yml`.
8. Нажмите **Commit changes**.

После загрузки сборка должна запуститься автоматически. Если она не началась:

1. Откройте вкладку **Actions**.
2. Слева выберите **Build Android APK**.
3. Нажмите **Run workflow**, затем ещё раз **Run workflow**.

## Как скачать APK

1. Во вкладке **Actions** откройте завершившийся запуск с зелёной галочкой.
2. Внизу страницы найдите раздел **Artifacts**.
3. Скачайте `CubeDodge-Android-APK`.
4. Распакуйте скачанный ZIP. Внутри будет
   `CubeDodge-android-arm64-debug.apk`.
5. Передайте APK на телефон и откройте его. Android может попросить разрешить
   установку приложений из этого источника.

## DualSense

Геймпад подключается средствами Android до запуска игры:

- Bluetooth: удерживайте **Create + PS**, пока подсветка не начнёт мигать,
  затем выберите контроллер в настройках Bluetooth телефона;
- USB: подключите DualSense кабелем USB-C с поддержкой передачи данных.

При первом USB-подключении Android покажет системный запрос доступа к
DualSense — его нужно разрешить. Приложение не забирает интерфейс управления:
стики и кнопки продолжает читать Android, а отдельный USB-канал используется
только для адаптивных триггеров. Если прошивка телефона запретит такие HID-
команды, обычное управление и вибрация продолжат работать, но адаптивного
сопротивления не будет.

## Управление настройками

- **Options** — открыть или закрыть оверлей настроек;
- крестовина — выбирать пункт;
- **Square** — включить или выключить выбранный пункт; пока меню открыто,
  эта кнопка не расходует деш;
- **L1 + R1** — ускорение, только если оно заранее включено в настройках.

На ПК оверлей можно открыть клавишей **F10**, переключить пункт — **Enter**,
а включённое ускорение проверить удержанием **Space**.

После первого запуска нужно проверить фактическую раскладку кнопок на телефоне.
Разные версии Android иногда по-разному называют оси триггеров; если L2/R2 или
одна из кнопок определится неправильно, это исправляется небольшой правкой
маппинга без изменения игровой логики.

## Повторная сборка

После изменения `upload/main(1).cpp` загрузите новую версию файла в репозиторий.
GitHub автоматически соберёт новый APK. Старые Artifacts хранятся 30 дней.
